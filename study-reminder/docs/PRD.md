# 产品需求文档 (PRD)

**项目名称**：StudyReminder（学习待办提醒工具）  
**版本**：v1.0  
**日期**：2026-04-24  
**作者**：AI 助手（代笔）  

---

## 1. 背景与目标

### 1.1 背景
用户是一名学生/自学者，每天有不同类型的学习任务（LeetCode、英语听力、机器学习等），这些任务的时间属性和紧迫性各不相同。现有的 Todo 软件过于通用或过于复杂，用户需要一个**轻量化、常驻桌面、针对学习场景**的待办提醒工具。

### 1.2 目标
- 提供一个 Windows 桌面应用，开机自启，半隐藏悬浮在桌面边缘。
- 支持任务的增、查、改、删，以及到期自动通知。
- 任务分为**月视图**（宏观浏览）和**日视图**（精细时间安排）。
- 支持周期性任务（每周多天），截止日期提醒和逾期二次提醒。
- 数据本地存储，预留未来同步和 AI 接口。

## 3. 功能范围

### 3.1 v1.0 核心功能
| 功能 | 描述 |
|------|------|
| 任务 CRUD | 支持创建、查看、编辑、删除任务 |
| 月视图 | 顶部显示跨月大任务的横幅，每日格子显示该日任务简要列表，截止日任务标注“DDL: 日期” |
| 日视图 | 左侧时间轴显示具体时间段任务（块状横条），下方/右侧无时间任务按优先级排列 |
| 周期性任务 | 按星期几重复（支持多选），如每周二、四 |
| 系统通知 | 具体时间任务提前5分钟弹出；截止日当天弹出；逾期未完成再次弹出（每天一次） |
| 半隐藏悬浮 | 默认吸附在屏幕边缘，鼠标悬停展开显示当天任务概况，点击进入大界面 |
| 开机自启 | 自动注册到 Windows 启动项 |
| 全部任务视图 | 所有未完成任务按综合排序（紧迫度+优先级）展示 |
| 优先级 | 高/中/低三级，颜色标记（红/黄/灰） |

### 3.2 后续版本（预留）
| 功能 | 描述 |
|------|------|
| 手机同步 | 通过轻量后端或 Supabase 实现数据同步 |
| 自然语言添加 | 输入“明天下午3点做听力”自动解析并创建 |
| AI 智能排序 | 根据任务内容和截止日建议优先级 |
| 日历拖拽 | 在月/日视图中拖拽任务调整日期/时间 |
| 标签/分类管理 | 自定义标签并过滤 |

## 4. 用户界面草图（文字描述）

### 4.1 整体布局
- **常规状态**：一个约 40x40px 的圆形图标（或窄条）吸附在屏幕右侧边缘。
- **鼠标悬停**：图标向左侧展开为约 300x600px 的小窗口，显示当天任务紧凑列表（标题+时间+截止标记）。
- **点击**：打开主窗口（约 900x600px），默认展示**月视图**。

### 4.2 主窗口 - 月视图
```
┌─────────────────────────────────────────────┐
│  < 2025年4月 >    [日视图]  [全部任务]       │
├─────────────────────────────────────────────┤
│  ████████████ (跨月任务横幅) ████████████    │
├─────┬─────┬─────┬─────┬─────┬─────┬───────┤
│ 周一 │ 周二 │ 周三 │ 周四 │ 周五 │ 周六 │ 周日 │
├─────┼─────┼─────┼─────┼─────┼─────┼───────┤
│  1   │  2   │  3   │  4   │  5   │  6   │  7   │
│ LeetCode │ 听力  │      │      │      │      │       │
│ DDL:4/5  │      │      │      │      │      │       │
├─────┼─────┼─────┼─────┼─────┼─────┼───────┤
│ ...  │ ...  │ ...  │ ...  │ ...  │ ...  │ ...  │
└─────┴─────┴─────┴─────┴─────┴─────┴───────┘
```

- 每日格子内最多显示 3 条任务（超出用“+X 更多”折叠）。
- 截止日任务名称后跟带红点的“DDL: 4/5”。
- 跨月任务在顶部显示为渐变横幅，标注任务名和起止日期。

### 4.3 主窗口 - 日视图（点击某一天进入）
```
┌─────────────────────────────────────────────┐
│  2025年4月10日 周四    [月视图]  [全部任务]  │
├─────────────────────────────────────────────┤
│  全天任务                          (优先级) │
│  ┌─────────────────────────────────┐        │
│  │ 英语听力    (截止:4/12 逾期!)   │  🔴高  │
│  └─────────────────────────────────┘        │
│                                             │
│  时间安排                                   │
│  14:00 ┌────────────────────┐              │
│        │  LeetCode          │              │
│  15:00 └────────────────────┘              │
│                                             │
│  未安排时间                                 │
│  ┌─────────────────────────────────┐        │
│  │ 整理笔记              (优先级中) │ 🟡    │
│  └─────────────────────────────────┘        │
└─────────────────────────────────────────────┘
```

- 全天任务区：包含截止日期的任务，逾期标红。
- 时间安排区：用时间轴左侧显示时段，右侧横条表示任务时长。
- 未安排时间区：按优先级从高到低排列。

## 5. 非功能需求

| 编号 | 需求 |
|------|------|
| NFR1 | 开机自启时间 ≤ 5秒（应用启动至背景通知可用） |
| NFR2 | 常驻内存 ≤ 100 MB（空闲状态） |
| NFR3 | 打包体积 ≤ 30 MB（v1.0） |
| NFR4 | 任务数据存储为本地 SQLite 文件，导出为 JSON 方便备份 |
| NFR5 | 系统通知使用 Windows Toast（无需用户额外配置） |
| NFR6 | 所有 UI 响应时间 ≤ 200ms（鼠标悬停展开、切换视图、保存任务等） |

## 6. 技术约束与扩展预留

- **本地优先**：v1.0 不依赖网络，所有数据存储在本地。
- **数据导出接口**：SQLite 文件路径暴露给用户，或提供“导出为 JSON”功能。
- **AI 扩展**：保留一个可配置的 HTTP 端点，后续可接入任何 LLM API。当前阶段先定义好命令通信协议（如 `ai.interpret` 接受自然语言文本，返回结构化任务数据）。
- **同步扩展**：数据库层设计时预留 `sync_version`、`last_synced_at` 字段，便于后续同步冲突处理。

## 7. 验收标准

| 验收条件 | 描述 |
|----------|------|
| AC1 | 用户启动应用后，悬浮图标出现在屏幕右侧，鼠标悬停显示当天任务列表 |
| AC2 | 点击悬浮图标进入主窗口，默认月视图，顶部横幅显示跨月任务，每日格子显示任务 |
| AC3 | 点击某一天进入日视图，显示时间段任务和未安排时间任务 |
| AC4 | 用户可以新建、编辑、删除任务，填写所有字段 |
| AC5 | 周期性任务（如每周二、四）在指定日期自动出现 |
| AC6 | 具体时间任务提前5分钟弹出 Windows 通知；截止日当天弹出通知；逾期未完成每日通知一次 |
| AC7 | 应用在 Windows 设置中注册为开机自启（非强制，可取消） |
| AC8 | 全部任务视图按紧迫度+优先级排序显示，未完成在前 |
